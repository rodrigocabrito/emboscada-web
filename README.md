# Emboscada — Plataforma da Equipa

Internal web platform for **Emboscada Parque Aventura** (a paintball / laser tag / adventure park). It's a staff-only tool for managing game sessions, the team, monitor scheduling, ammunition stock, evaluations, and internal announcements.

## What it does

- **Authentication & roles** — email/password login (Firebase Auth) with three staff roles:
  - **Administrador** — full access to everything.
  - **Monitor Líder** — can edit session line items (packs/extras/others) and register payments; can be scheduled like a monitor.
  - **Monitor** — can edit session line items; session data is view-only.
- **Início (Dashboard)** — each user's upcoming assigned sessions at a glance.
- **Sessões** — calendar view, full session detail (client info, players, type, caliber, packs/extras/others, live financial summary and payment flow).
- **Disponibilidades** — monthly availability calendar (full / partial with time limits), including Portuguese public holidays.
- **Equipa** — team directory grouped by role, with tenure achievements.
- **Comunicados** — internal announcements with emoji reactions; admins publish/edit, everyone reacts.
- **Perfil** — personal info, password change, and the user's own evaluation summary.
- **Admin panel**
  - **Utilizadores** — create users (with a welcome email), promote/demote monitors ↔ monitor leaders, delete users.
  - **Sessões** — filterable/sortable table of all sessions.
  - **Catálogo** — manage packs, activities and prices.
  - **Stock** — ammunition stock per caliber (.50 / .68) with restocks, manual counts, removals and per-session usage tracking.
  - **Escala** — assign monitors to a day's sessions based on their availability, with scheduling-rule warnings.
- **Email notifications** — welcome email on user creation, notification to all users when a comunicado is published, and a notification to a user when their evaluation is updated.

## Tech stack

- **Frontend:** React 19, Vite, React Router, TanStack Query
- **Backend-as-a-service:** Firebase (Authentication + Cloud Firestore)
- **Serverless:** Vercel Functions (`/api`) for sending email — token-verified with `jose`, delivered via Gmail SMTP (`nodemailer`)
- **Hosting:** Vercel

## Running locally

### Prerequisites
- **Node.js 20+** and npm
- A **Firebase** project with Authentication (Email/Password) and Cloud Firestore enabled

### 1. Install
```bash
npm install
```

### 2. Configure environment
Create a `.env.local` file in the project root with your Firebase web-app config:

```bash
VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=...
VITE_FIREBASE_PROJECT_ID=...
VITE_FIREBASE_STORAGE_BUCKET=...
VITE_FIREBASE_MESSAGING_SENDER_ID=...
VITE_FIREBASE_APP_ID=...

# Optional: public base URL used for links/assets inside emails
# VITE_PUBLIC_URL=https://your-app.vercel.app
```

### 3. Start the dev server
```bash
npm run dev
```
The app runs at the URL Vite prints (default `http://localhost:5173`).

> **Note:** the email feature lives in `/api/send-email` (a Vercel serverless function). It does **not** run under the plain `vite` dev server — use `vercel dev` locally or a Vercel deployment to exercise it.

## Email (serverless function)

Sending email requires these **server-side** environment variables (set them in Vercel → Project → Settings → Environment Variables, never in client code):

| Variable | Purpose |
|----------|---------|
| `FIREBASE_PROJECT_ID` | Verifies the caller's Firebase ID token and admin role |
| `GMAIL_USER` | Gmail address used as the sender |
| `GMAIL_APP_PASSWORD` | Gmail **App Password** (requires 2-Step Verification) |

The function verifies the caller is an authenticated admin before sending, so the endpoint can't be abused.

## Security rules

Firestore security rules live in [`firestore.rules`](./firestore.rules). Publish them via **Firebase Console → Firestore Database → Rules → paste → Publish** (or `firebase deploy --only firestore:rules` if you set up the Firebase CLI). They require an authenticated staff account for all access and restrict sensitive collections to admins.

## Deployment

The app is designed for **Vercel**: connect the repository, add the `VITE_*` and email environment variables, and deploy. The `/api` directory is automatically served as serverless functions.
